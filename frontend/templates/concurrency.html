🔒 Concurrency Control & Recovery

ACID · Isolation Levels · Savepoints · Write-Ahead Log
Atomicity
All-or-nothing. Either all ops in a transaction commit, or all are rolled back.
Consistency
Each transaction takes DB from one valid state to another, obeying all constraints.
Isolation
Concurrent transactions don't see each other's intermediate states.
Durability
Committed transactions persist even through system crash (WAL / redo log).
💳 SERIALIZABLE Transaction Demo
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE; BEGIN; SELECT total_fare FROM booking WHERE booking_id=? FOR UPDATE; UPDATE booking SET total_fare = total_fare - amount WHERE booking_id=?; UPDATE booking SET total_fare = total_fare + amount WHERE booking_id=?; COMMIT;
💾 SAVEPOINT Demo
BEGIN; UPDATE flight SET flight_status=? WHERE flight_id=?; SAVEPOINT sp1; UPDATE flight SET flight_status=? WHERE flight_id=?; -- Option A: ROLLBACK TO SAVEPOINT sp1 (partial undo) -- Option B: COMMIT (both updates)
🔄 Recovery Simulation — Write-Ahead Log (WAL)
-- WAL Concept: -- 1. Log BEFORE state to undo-log -- 2. Apply change (dirty write) -- 3A. COMMIT → flush redo log → durable -- 3B. CRASH → ROLLBACK → restore from undo-log
